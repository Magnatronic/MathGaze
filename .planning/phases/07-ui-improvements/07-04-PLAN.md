<objective>
Add the object list panel to the right rail (Select tool + nothing selected state) and fix first-click snapping for Line, Circle, and Protractor two-point mode.

Purpose: The object list solves a real daily frustration — when geometry objects overlap or are small, the student must click precisely on the canvas. The list provides a gaze-friendly alternative requiring no precision. First-click snap is a consistency fix — the snap ring already tracks the cursor on first click via ghost preview, but the commit was ignoring that snap point.

Output: ObjectListItem class; ObjectList ObservableCollection on RightRailViewModel; HasObjectList computed property; ObjectListPanel in RightRail.xaml; first-click snap added to Line/Circle/Protractor cases in ToolViewModel.HandleCanvasClick.

**Also update HandleMouseMove to show snap ring during DrawState.Idle when a drawing tool is active (D-23 enhancement — snap ghost shows before first click):**

Current HandleMouseMove only runs snap during `DrawState.AnchorPlaced`. The Idle state sets `LastSnap = null`. Update to also compute snap during Idle for drawing tools:

```csharp
public void HandleMouseMove(SKPoint screenPx, CoordinateMapper mapper, SnapEngine snap)
{
    GhostCursorPx = screenPx;

    if (DrawState == DrawState.AnchorPlaced)
    {
        if (ActiveTool == ToolMode.Protractor)
        {
            // Protractor: no snap during placement; ghost tracks cursor freely
            LastSnap = null;
            StatusMessage = AnchorLine is null
                ? "Click to set baseline direction"
                : "Click 2nd line";
        }
        else
        {
            var result = snap.Snap(screenPx, _geometryService.Objects, mapper);
            LastSnap = result;
            StatusMessage = result.Label is not null
                ? $"Click 2nd point · snap: {result.Label}"
                : (ActiveTool == ToolMode.Circle ? "Click radius point" : "Click 2nd point");
        }
    }
    else if (DrawState == DrawState.Idle &&
             ActiveTool is ToolMode.Line or ToolMode.Circle)
    {
        // Show snap ring during Idle for drawing tools — so student sees snap feedback
        // before placing first click (D-23 enhancement)
        var result = snap.Snap(screenPx, _geometryService.Objects, mapper);
        LastSnap = result;
    }
    else
    {
        LastSnap = null;
    }

    GhostChanged?.Invoke(this, EventArgs.Empty);
}
```

This adds snap tracking for Line and Circle while in Idle state. Protractor Idle already handles snap differently (it hit-tests for line objects). Select, Point, and Text tools don't need snap preview during Idle.

  <files>MathGaze/ViewModels/PdfCanvasViewModel.cs</files>
  <read_first>
    - MathGaze/ViewModels/PdfCanvasViewModel.cs — read the full DrawGhostPreview method (starts around line 238). Identify:
      (a) the early-return gate `if (_toolVm.DrawState != DrawState.AnchorPlaced) return;`
      (b) the snap ring drawing block (the section starting with "Draw snap ring indicator")

  <action>
    Root cause: `DrawGhostPreview` returns early at the top if `DrawState != AnchorPlaced`.
    The snap ring rendering block is after this early return, so it is never reached in Idle state.
    The ToolViewModel DOES compute `LastSnap` during Idle hover for Line/Circle tools (lines 362-368
    in ToolViewModel.cs), but the render side never uses it.

    The fix is to add an early Idle-snap path BEFORE the `AnchorPlaced` early return. When
    `DrawState == Idle` and `ActiveTool` is Line or Circle, draw ONLY the snap ring (no anchor
    dot, no ghost line — there is no anchor yet), then return.

    In `PdfCanvasViewModel.cs`, modify `DrawGhostPreview` as follows:

    1. Locate the method signature: `private void DrawGhostPreview(SKCanvas canvas)`
    2. After the `if (_coordinateMapper is null) return;` guard (first line), add a new block
       BEFORE the `if (_toolVm.DrawState != DrawState.AnchorPlaced) return;` line:

    ```csharp
    // Idle-state snap ring: show snap feedback before first click for Line and Circle tools (Gap 8).
    // DrawState is Idle but ToolViewModel.HandleMouseMove has computed LastSnap.
    // No anchor, no ghost line — just the snap ring to signal "snap available here".
    if (_toolVm.DrawState == DrawState.Idle &&
        (_toolVm.ActiveTool == ToolMode.Line || _toolVm.ActiveTool == ToolMode.Circle) &&
        _toolVm.LastSnap.HasValue)
    {
        float ds = (float)(_dpiScale * _mainVm.ZoomFactor);
        bool isSnapped = _toolVm.LastSnap.Value.Label is not null;
        var ringCenter = _toolVm.LastSnap.Value.Position;

        using var idleSnapPaint = new SKPaint
        {
            Style       = SKPaintStyle.Stroke,
            Color       = isSnapped
                ? new SKColor(0x3B, 0x6F, 0xD4, 220)   // solid cobalt ring at snap point
                : new SKColor(0x3B, 0x6F, 0xD4, 80),    // faint ring at cursor (no snap nearby)
            StrokeWidth = 2f * ds,
            IsAntialias = true,
            PathEffect  = isSnapped
                ? SKPathEffect.CreateDash(new float[] { 3f, 3f }, 0f)  // dashed = snapped
                : null,
        };
        canvas.DrawCircle(ringCenter, 18f * ds, idleSnapPaint);

        if (isSnapped)
        {
            using var idleSnapDotPaint = new SKPaint
            {
                Style       = SKPaintStyle.Fill,
                Color       = new SKColor(0x3B, 0x6F, 0xD4, 200),
                IsAntialias = true,
            };
            canvas.DrawCircle(ringCenter, 5f * ds, idleSnapDotPaint);
        }
        return;  // Do not draw anchor ring — there is no anchor in Idle state
    }
    ```

    3. Leave the existing `if (_toolVm.DrawState != DrawState.AnchorPlaced) return;` gate
       unchanged after the new block. The new block handles Idle; the gate handles all other
       non-AnchorPlaced states (Select, Text, Protractor in Idle, etc.).

    4. Do NOT modify the existing AnchorPlaced snap ring block (lines ~299-333). It remains
       as the mid-draw snap ring for the second click.

    Key considerations:
    - When `LastSnap.HasValue` is false (cursor is over empty canvas far from any snap points),
      the new block is skipped entirely and the canvas shows no ring — clean, no visual noise.
    - When `LastSnap.HasValue` is true but `Label` is null (cursor within 20px of a snap point
      but the SnapEngine found no exact snap), the faint ring (alpha=80) renders at the cursor.
      This gives the student a soft "you're close" indicator.
    - When snapped (`Label` is not null), the dashed cobalt ring + filled dot renders at the
      exact snap point, giving clear "will snap here" feedback before first click.
    - The `return` at the end of the block prevents the `AnchorPlaced` section from running
      (there is no anchor in Idle, so nothing else to draw).
  </action>

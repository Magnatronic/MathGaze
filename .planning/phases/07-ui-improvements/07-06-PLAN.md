    ## Part A: Settings auto-close on tool activation

    When the settings panel is open and the user clicks a tool button, the settings panel should
    close. The ToolViewModel fires `PropertyChanged` for `ActiveTool` when any tool is activated
    (all six `ActivateXxx` commands set `ActiveTool`).

    Implementation: In `MainViewModel`, subscribe to `_toolVm.PropertyChanged` in the constructor.
    When `ActiveTool` changes and `_settingsVm.IsSettingsPanelOpen` is true, call
    `_settingsVm.CloseSettings()`.

    In `MainViewModel.cs`:
    1. At the end of the constructor (after setting `_settingsVm = settingsViewModel`), add:
       ```csharp
       _toolVm.PropertyChanged += OnToolPropertyChanged;
       ```
    2. Add the handler method (private, inside the class):
       ```csharp
       private void OnToolPropertyChanged(object? sender, System.ComponentModel.PropertyChangedEventArgs e)
       {
           if (e.PropertyName == nameof(ToolViewModel.ActiveTool) && _settingsVm.IsSettingsPanelOpen)
               _settingsVm.CloseSettings();
       }
       ```
    3. Add the required `using` if not already present:
       `using System.ComponentModel;` (check â€” likely already present or not needed since the param type is fully qualified)

    Note: `SettingsViewModel.CloseSettings()` is a private method decorated with `[RelayCommand]`
    which generates `CloseSettingsCommand`. The method itself is private. Instead, set the property
    directly: `_settingsVm.IsSettingsPanelOpen = false;` â€” or call the generated command:
    `_settingsVm.CloseSettingsCommand.Execute(null);`
    Use `_settingsVm.IsSettingsPanelOpen = false;` â€” direct property assignment is the cleanest
    approach since both objects are in the same assembly and IsSettingsPanelOpen has a public setter
    (generated by `[ObservableProperty]`).

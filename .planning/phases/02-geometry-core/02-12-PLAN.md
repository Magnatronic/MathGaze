<objective>
Fix GAP-12: horizontal (0°/180°) orientation snap does not trigger in the running app, while vertical (90°) and 45° snaps work correctly.

Root cause analysis:

Looking at SnapEngine.cs section 3 (orientation guides, lines 69-103), the horizontal check at lines 74-80 is:

```csharp
float dH = Math.Abs(cursorPx.Y - snapPx.Y);
if (dH < bestDist)
{
    var candidate = new SKPoint(cursorPx.X, snapPx.Y);
    float d = SKPoint.Distance(cursorPx, candidate);
    if (d < bestDist) { bestDist = d; best = candidate; label = "horizontal"; }
}
```

The outer guard `dH < bestDist` checks against `bestDist` which starts at `SnapThresholdPx = 20f` but may have been REDUCED by prior endpoint snaps (section 1) or intersection snaps (section 2). If a nearby object endpoint already set `bestDist` to e.g. 8f, horizontal snap only triggers when `dH < 8` — a tighter window than the nominal 20px threshold.

For vertical (lines 81-88) and 45° (lines 89-100), the same issue exists in principle, but they are less likely to be suppressed in practice because:
- Vertical snap aligns cursor's X to a snap point's X — a fundamentally different axis than endpoint proximity (which is measured diagonally).
- 45° snap uses a fundamentally different metric (`|dx| ≈ |dy|`) that happens to avoid the bestDist suppression in common test cases.

The definitive fix: add an explicit `SnapThresholdPx` guard to the horizontal, vertical, and 45° checks so they test BOTH (a) within the orientation threshold and (b) within bestDist. This mirrors the logic of section 1 (endpoint snap) which also uses `if (d < bestDist)` where `bestDist` starts at `SnapThresholdPx`. With the fix, orientation guides are never suppressed by a prior endpoint snap at the same or adjacent position.

Additionally, the plan 02-11 creates `SnapEngineTests.cs` (if it has not already been created). This plan builds on that by ensuring horizontal snap tests pass and adding further edge-case tests for robustness.

Purpose: Make horizontal orientation snap reliable by adding the absolute threshold guard to all three orientation guide checks.
Output: Modified `SnapEngine.cs` with corrected horizontal (and defensive vertical/45°) guards. Verified by unit tests in `SnapEngineTests.cs`.
</objective>

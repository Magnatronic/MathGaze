<objective>
Fix three blocking UAT gaps in the geometry layer:

1. GAP-1/GAP-2 (same root cause): Committed geometry objects and the ghost preview cursor both
   appear offset from the actual gaze cursor position. Root cause: `LoadCurrentPageAsync` renders
   the PDF bitmap WITHOUT applying `_dpiScale` to its pixel dimensions, while `EnsureCoordinateMapper`
   DOES include `_dpiScale` in its scale formula. At any DPI above 96 (PixelsPerDip > 1.0), the
   bitmap is rendered smaller than the coordinate space expects, producing a systematic offset between
   the bitmap origin and where CoordinateMapper thinks geometry should appear.

2. GAP-3: Up nudge moves object down; Down nudge moves object up. Root cause: the Y nudge delta sign
   convention is inverted. PDF coordinate space has Y increasing UPWARD, so "move up on screen" means
   INCREASING YPt. `NudgeUp` currently passes `-NudgeStepPx` (decreases YPt = moves DOWN). Fix: swap
   the signs in NudgeUp and NudgeDown.

Purpose: Geometry objects must appear where the student gazes — this is the core product promise.
Output: Modified PdfCanvasViewModel.cs and RightRailViewModel.cs.
</objective>

EnsureCoordinateMapper scale formula (CORRECT — includes _dpiScale):
```csharp
double scale = (_dpiScale * 96.0 / 72.0) * _mainVm.ZoomFactor;
```

LoadCurrentPageAsync scale formula (BUG — missing _dpiScale):
```csharp

```

NudgeUp/NudgeDown (BUG — Y sign inverted relative to PDF Y-axis):
```csharp
[RelayCommand(CanExecute = nameof(CanNudge))]
private void NudgeUp()   => DispatchNudge(0, -NudgeStepPx);  // ← BUG: negative = decrease YPt = moves DOWN in PDF space
[RelayCommand(CanExecute = nameof(CanNudge))]
private void NudgeDown() => DispatchNudge(0, +NudgeStepPx);  // ← BUG: positive = increase YPt = moves UP in PDF space

// PDF Y-axis: 0 = bottom, positive = upward.
// PageToScreen: screenY = (pageHeightPt - yPt) * Scale + canvasOriginY
// So increasing yPt DECREASES screenY (moves UP on screen). ← confirmed in CoordinateMapper.cs
```

<objective>
Fix two remaining rendering correctness issues in the geometry placement pipeline.

**GAP-6 — Intermittent placement inaccuracy:**

Root cause identified: In `PdfCanvas.xaml.cs ReportCanvasSize()`, the call order is:
  1. `_vm.SetCanvasSize(widthPx, heightPx)`   ← triggers async LoadCurrentPageAsync + CoordinateMapper creation
  2. `_vm.SetDpiScale(dpiInfo.PixelsPerDip)`  ← sets _dpiScale used by CoordinateMapper

`SetCanvasSize` calls `LoadCurrentPageAsync` (fire-and-forget async) and also `_mainVm.OnCanvasSizeChanged()` which can set `ZoomFactor`, which fires `OnMainViewModelPropertyChanged`, which calls `LoadCurrentPageAsync` again — all of this before `SetDpiScale` has run. During this window, `_dpiScale` may still hold a stale value (e.g., 1.0 from initialisation at a 1.5-DPI monitor), causing `EnsureCoordinateMapper` to compute a wrong `scale`, placing the bitmap and geometry at mismatched positions.

Fix: swap the call order in `ReportCanvasSize` so `_vm.SetDpiScale(dpiInfo.PixelsPerDip)` is called BEFORE `_vm.SetCanvasSize(widthPx, heightPx)`. This guarantees `_dpiScale` is always correct before any CoordinateMapper creation or bitmap load is triggered.

**GAP-7 — Snap ring flickers during mid-draw:**

Root cause: `DrawGhostPreview` only draws the snap ring indicator when `_toolVm.LastSnap?.Label is not null`. `LastSnap.Label` is null whenever the cursor is NOT within 20px of a snap candidate. So the ring disappears entirely when moving through free space and reappears at snap zones — the "flicker" is actually the ring appearing and vanishing repeatedly.

Fix: Change the snap ring rendering so:
- When `DrawState == AnchorPlaced` (we are in a draw operation), ALWAYS draw a cursor-tracking ring at `GhostCursorPx`
- When `Label is not null` (a snap candidate is nearby), ADDITIONALLY draw the larger snap indicator ring at `LastSnap.Position` with its current styling
- This gives continuous visual feedback during draw: the cursor ring is always there; the snap ring appears on top when snapping

The cursor-tracking ring uses a smaller, lighter style (18px radius, 60% opacity, no dash) to distinguish it from the snap ring (18px radius, 80% opacity, dashed). Or more simply: when snapped, show the snap ring at `LastSnap.Position`; when not snapped, show the ring at `GhostCursorPx` with lighter opacity. Either approach produces continuous ring visibility.

Purpose: Correct coordinate pipeline eliminates inconsistent point placement. Continuous snap ring eliminates confusing disappearing feedback during draw operations.
Output: PdfCanvas.xaml.cs with corrected SetDpiScale call order; PdfCanvasViewModel.cs with snap ring that tracks cursor continuously during draw.
</objective>

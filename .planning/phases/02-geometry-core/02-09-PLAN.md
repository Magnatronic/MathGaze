<objective>
Fix geometry state persisting across PDF sessions (GAP-10).

Root cause: `MainViewModel.OpenFileAsync` opens a new PDF and calls `_pdfCanvasVm.OnDocumentOpenedAsync()` to start rendering, but never calls `_geometryService.Reset()`. All `GeometryObject` instances from the previous document remain in `GeometryService._objects` and are rendered on top of the new PDF's first page.

`GeometryService.Reset()` already exists and correctly clears `_objects`, calls `_undoService.Clear()`, and raises `ObjectsChanged` so the canvas repaints. It just needs to be called from the PDF load path.

The fix is a single method call: add `_geometryService.Reset()` inside `OpenFileAsync` in `MainViewModel`, on the UI thread (within the `InvokeAsync` block), BEFORE `OnDocumentOpenedAsync()` is called so the canvas is clean when the first page renders.

`MainViewModel` does not currently hold a reference to `IGeometryService`. This reference must be added via constructor injection.

Purpose: Gaze students who open a second exam paper must not see geometry from a previous paper. A ghost object overlaid on printed text creates ambiguity — critical for exam integrity.
Output: MainViewModel with IGeometryService constructor parameter; Reset() called on every PDF open.
</objective>

The block after change must read:
```csharp
await System.Windows.Application.Current.Dispatcher.InvokeAsync(() =>
{
    FileName      = Path.GetFileName(filePath);
    TotalPages    = _pdfService.PageCount;
    CurrentPage   = 1;
    IsPdfOpen     = true;
    ScrollOffsetY = 0;
    _geometryService.Reset();
});
```

Placing Reset() inside InvokeAsync ensures it runs on the UI thread (same thread as the geometry and canvas render loop), preventing any cross-thread ObjectsChanged event firing. Do NOT call Reset() outside InvokeAsync or before the success check.

No other changes to MainViewModel.cs.

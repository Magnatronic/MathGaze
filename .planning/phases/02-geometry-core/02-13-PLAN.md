<objective>
Fix GAP-13: geometry objects drawn on page N remain visible when the student navigates to page N+1 within the same PDF.

Root cause: `MainViewModel.OnCurrentPageChanged` (the CommunityToolkit.Mvvm-generated partial method that fires when `CurrentPage` changes) does not call `_geometryService.Reset()`. GAP-10 added `Reset()` only in `OpenFileAsync` (new PDF open). Page navigation is a separate code path through `PreviousPage()` / `NextPage()` → `CurrentPage--` / `CurrentPage++` → `OnCurrentPageChanged`.

Looking at MainViewModel.cs line 133-140:

```csharp
partial void OnCurrentPageChanged(int value)
{
    OnPropertyChanged(nameof(PageLabel));
    PreviousPageCommand.NotifyCanExecuteChanged();
    NextPageCommand.NotifyCanExecuteChanged();
}
```

No `Reset()` call here. The fix: add `_geometryService.Reset()` inside `OnCurrentPageChanged`.

Design considerations:
- GAP-10 fix (line 256): `_geometryService.Reset()` in `OpenFileAsync` inside `Dispatcher.InvokeAsync`. This is necessary because `OpenFileAsync` sets `CurrentPage = 1` (line 253) which triggers `OnCurrentPageChanged` → if `OnCurrentPageChanged` also calls `Reset()`, Reset() would be called TWICE during PDF open (once from page change, once from OpenFileAsync). This is harmless because `Reset()` is idempotent (clearing an already-empty service is a no-op), but it's redundant.
- The cleaner fix: call `Reset()` ONLY from `OnCurrentPageChanged`, and remove the explicit `Reset()` call from `OpenFileAsync`. But that changes more code and risks breaking the GAP-10 fix. The safe approach: ADD `Reset()` to `OnCurrentPageChanged`, KEEP the existing `Reset()` in `OpenFileAsync`. Two calls during PDF open is harmless.
- `IGeometryService.Reset()` clears both the object list AND the undo/redo stacks. This is the correct behavior for page navigation — each page is an independent session.

Purpose: Ensure each page navigation clears the geometry canvas, giving each exam page an independent drawing surface.
Output: Modified `MainViewModel.cs` — `OnCurrentPageChanged` calls `_geometryService.Reset()`.
</objective>

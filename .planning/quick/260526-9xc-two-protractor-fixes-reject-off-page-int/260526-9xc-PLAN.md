### Task 1 — Reject off-page intersection

**File:** `MathGaze/ViewModels/ToolViewModel.cs`
**Case:** `(ToolMode.Protractor, DrawState.AnchorPlaced)`

Replace the silent `Math.Clamp` of intersection coords with a bounds check.
If `interPt` falls outside page bounds (±20pt margin), show status message
"Lines don't intersect on this page — extend lines to create a crossing"
and break (keeping AnchorPlaced so student can try a different second line).

(no clamping needed once placement is validated).

### Task 2 — Same-line second click → nearest endpoint

**File:** `MathGaze/ViewModels/ToolViewModel.cs`
**Case:** `(ToolMode.Protractor, DrawState.AnchorPlaced)`

Replace the `if (line2.Id == AnchorLine.Id) break;` bail with endpoint placement:

1. Compute screen positions of both endpoints
2. Pick nearest to `screenPx` as center (compare squared distances)
3. Baseline angle = `atan2(ep2Screen.Y - ep1Screen.Y, ep2Screen.X - ep1Screen.X)`
4. Flip if `cos(baselineAngle) < 0` so arc faces upward on screen
5. Place `ProtractorObject` with same line ID for both `line1Id`/`line2Id`
   (readout = 0° → suppressed by `< 0.5f` guard; student rotates freely)

**AppStyles.xaml â€” extend ToolTileStyle:**

The ToolTileStyle ControlTemplate currently only has IsMouseOver and IsPressed triggers.
WPF Button cannot use a DataTrigger inside ControlTemplate.Triggers to bind to the parent
ViewModel's ActiveTool (DataContext mismatch inside template). The correct pattern is a
Style-level DataTrigger that reads a Tag property set on each button.

Use a Tag-based pattern already established in this codebase (StepButtonStyle uses Tag='active'):

1. Add a new Style named `ToolTileActiveStyle` in AppStyles.xaml, `BasedOn="{StaticResource ToolTileStyle}"`, TargetType="Button".
2. Add a DataTrigger inside `ToolTileActiveStyle.Triggers` (Style-level, not ControlTemplate-level):
   `<DataTrigger Binding="{Binding Tag, RelativeSource={RelativeSource Self}}" Value="active">`
   Sets Background = BrushAccentSoft, BorderBrush = BrushAccent.

However, Tag is not bindable to ViewModel data directly in this pattern. The simpler correct
approach for WPF is: give each button a Tag that holds its ToolMode enum string, then bind the
Style via a converter â€” but that requires a converter.

**Simplest correct approach â€” use a MultiDataTrigger or a converter-free approach:**

The cleanest no-converter WPF approach: use a Style with a DataTrigger binding to
`ActiveTool` on the DataContext (ToolViewModel), comparing with the specific enum value
for each button. Each button gets its own inline Style that BasedOn ToolTileStyle and adds
ONE DataTrigger comparing `ActiveTool` to its specific ToolMode value.

**Implementation for ToolRail.xaml:**

For each of the 6 buttons (Select, Point, Line, Circle, Protractor, Text), add an inline
`<Button.Style>` block inside the `<Button>` element. Example for the Select button:

```xml
<Button.Style>
    <Style TargetType="Button" BasedOn="{StaticResource ToolTileStyle}">
        <Style.Triggers>
            <DataTrigger Binding="{Binding ActiveTool}" Value="Select">
                <Setter Property="Background"   Value="{StaticResource BrushAccentSoft}"/>
                <Setter Property="BorderBrush"  Value="{StaticResource BrushAccent}"/>
            </DataTrigger>
        </Style.Triggers>
    </Style>
</Button.Style>
```

The Value attribute for each button must match the ToolMode enum member name exactly:
- SelectButton    â†’ Value="Select"
- PointButton     â†’ Value="Point"
- LineButton      â†’ Value="Line"
- CircleButton    â†’ Value="Circle"
- ProtractorButtonâ†’ Value="Protractor"
- Text button     â†’ Value="Text"

No changes needed to AppStyles.xaml for this approach. The DataTrigger binding to
`{Binding ActiveTool}` resolves against the Button's DataContext, which is ToolViewModel
(set in code-behind for the ToolRail UserControl). This works without any converter because
WPF compares enum values by name when Value is a string in XAML.

**IMPORTANT: Enum namespace binding.** WPF DataTrigger Value="Select" (string) is compared
to the ToolMode enum using TypeConverter. This works in WPF for enums when the Value is
the exact enum member name as a string â€” no namespace prefix needed in the Value attribute.

**Apply to all 6 buttons in ToolRail.xaml**, adding the `<Button.Style>` block immediately
inside each `<Button>` element, before the `<StackPanel>` content.

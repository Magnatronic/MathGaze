**MainViewModel.cs** — three changes:

**A. ApplyFitPage** (line ~115): the zoom formula divides canvas pixels by `heightPt * (96.0/72.0)`, but must also divide by `_dpiScale` because `_canvasHeightPx` is in physical pixels while `heightPt * 96/72` is in logical pixels at 1× DPI. Fix:

```csharp
private void ApplyFitPage()
{
    if (!IsPdfOpen) return;
    var (_, heightPt) = _pdfService.GetPageDimensionsPt(CurrentPage - 1);
    if (heightPt <= 0) return;

    int canvasHeightPx = _pdfCanvasVm?.CanvasHeightPx ?? 0;
    if (canvasHeightPx <= 0) { ZoomFactor = 1.0; return; }

    double dpiScale = _pdfCanvasVm?.DpiScale ?? 1.0;
    // Physical canvas px = heightPt * (96/72) * dpiScale * zoom
    // Solve for zoom: zoom = canvasHeightPx / (heightPt * (96/72) * dpiScale)
    double zoom = canvasHeightPx / (heightPt * (96.0 / 72.0) * dpiScale);
    zoom = Math.Clamp(zoom, ZoomSteps[0], ZoomSteps[^1]);
    ZoomFactor = zoom;
}
```

**B. ClampScrollOffset** (line ~247): `pageHeightPx` uses `ZoomFactor * 96/72` — must include `dpiScale` to match physical canvas:

```csharp
private void ClampScrollOffset()
{
    if (!IsPdfOpen) return;
    var (_, heightPt) = _pdfService.GetPageDimensionsPt(CurrentPage - 1);
    double dpiScale = _pdfCanvasVm?.DpiScale ?? 1.0;
    double pageHeightPx = heightPt * (ZoomFactor * dpiScale * 96.0 / 72.0);
    double canvasH = _pdfCanvasVm?.CanvasHeightPx ?? 0;
    double maxScroll = Math.Max(0, pageHeightPx - canvasH);
    ScrollOffsetY = Math.Min(ScrollOffsetY, maxScroll);
    OnPropertyChanged(nameof(ScrollThumbTopRatio));
}
```

**D. ZoomIn cap** — replace the `ZoomIn` body to cap at the zoom where page width fills canvas width. After the `next > 0` check, clamp `next` so page never exceeds canvas width:

```csharp
[RelayCommand]
private void ZoomIn()
{
    _isFitPageMode = false;
    var next = ZoomSteps.FirstOrDefault(z => z > ZoomFactor + 0.001);
    if (next <= 0) return;

    // Cap at the zoom where page width exactly fills the canvas width (no H-overflow)
    if (IsPdfOpen && _pdfCanvasVm is not null)
    {
        var (widthPt, _) = _pdfService.GetPageDimensionsPt(CurrentPage - 1);
        double dpiScale = _pdfCanvasVm.DpiScale;
        int canvasW = _pdfCanvasVm.CanvasWidthPx;
        if (canvasW > 0 && widthPt > 0)
        {
            double maxZoom = canvasW / (widthPt * dpiScale * 96.0 / 72.0);
            next = Math.Min(next, maxZoom);
        }
    }

    if (next > ZoomFactor + 0.001)
        ZoomFactor = next;
}
```

The cached `SKPaint` and `SKFont` objects are created once with hardcoded sizes. At 2× DPI a 2.5px stroke renders as 1.25 logical points — visually hairline-thin. The fix is to pass `dpiScale` into `Draw()` and apply it when creating/updating the paint and font sizes.

Because the paints are cached as fields (never per-frame — existing performance pattern), the cleanest approach is to update their sizes at the top of `Draw()` when dpiScale changes. Add a `_lastDpiScale` field so we only update when DPI actually changes (avoids per-frame setter calls even though SKPaint setters are cheap).

**1. Add field:**
```csharp
private double _lastDpiScale = 0.0; // 0 forces first-run update
```

**2. Change `Draw` signature** to accept `dpiScale`:

```csharp
public void Draw(SKCanvas canvas, CoordinateMapper? mapper, double dpiScale = 1.0)
```

**3. At the top of `Draw()`, before the object loops, add:**
```csharp
if (Math.Abs(dpiScale - _lastDpiScale) > 0.001)
{
    _lastDpiScale = dpiScale;
    float s = (float)dpiScale;
    _normalPaint.StrokeWidth         = 2.5f * s;
    _selectedPaint.StrokeWidth       = 2.5f * s;
    _subRingActivePaint.StrokeWidth  = 2.5f * s;
    _tickMajorPaint.StrokeWidth      = 1.5f * s;
    _tickMinorPaint.StrokeWidth      = 1.0f * s;

}
```

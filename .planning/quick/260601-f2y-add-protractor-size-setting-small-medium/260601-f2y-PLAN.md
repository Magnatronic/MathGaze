# Quick Task 260601-f2y: Protractor Size + Rail Overflow Fix

## Goal

1. Let the student pick Small / Medium / Large protractor size in the Settings panel. New
   protractors use the selected size; existing saved ones are unaffected.
2. Fix the right rail overflow so the Delete button is always reachable (ScrollViewer).

## Size mapping (PDF points → screen px at zoom=1, 96 DPI, scale≈1.33)

| Size   | RadiusPt | Screen px |
|--------|----------|-----------|
| Small  |  80 pt   | ~107 px   |
| Medium | 144 pt   | ~192 px   | ← current default (backwards-compat)
| Large  | 200 pt   | ~267 px   |

## Tasks

### Task 1 — Add `RadiusPt` instance property to `ProtractorObject`

**Files:** `MathGaze/Core/Geometry/ProtractorObject.cs`

- Add `public double RadiusPt { get; set; } = DefaultRadiusPt;`
  JSON deserialization will default to `DefaultRadiusPt` (144) for existing saved sessions.
- Update `HitTest` to use `this.RadiusPt` instead of `DefaultRadiusPt`.

### Task 2 — Add `ProtractorSize` to `UserPreferences`

**Files:** `MathGaze/Properties/UserPreferences.cs`

- Add `ProtractorSize` string property (default `"Medium"`) to `PrefsData`.
- Add static `ProtractorSizeRadiusPt` property: "Small"→80.0, "Large"→200.0, else 144.0.

### Task 3 — Set `RadiusPt` on new protractors in `ToolViewModel`

**Files:** `MathGaze/ViewModels/ToolViewModel.cs`

- Both creation sites (line-intersection path ~L288, two-point path ~L308):
  `protractor.RadiusPt = UserPreferences.ProtractorSizeRadiusPt;`
  Set this before `ExecuteCommand`.

### Task 4 — Update renderer to use `obj.RadiusPt`

**Files:** `MathGaze/ViewModels/GeometryLayerViewModel.cs`

- In `DrawProtractor`: change `ProtractorObject.DefaultRadiusPt` → `obj.RadiusPt`
  (the proxy-point edge calculation at ~L376).

### Task 5 — Add size state + command to `SettingsViewModel`

**Files:** `MathGaze/ViewModels/SettingsViewModel.cs`

- Add three `[ObservableProperty]` booleans: `_isProtractorSmall`, `_isProtractorMedium`,
  `_isProtractorLarge`. Initialise from `UserPreferences.ProtractorSize`.
- Add `SetProtractorSize(string size)` RelayCommand: set the three booleans, update
  `UserPreferences.ProtractorSize`, call `UserPreferences.Save()`.

### Task 6 — Add PROTRACTOR SIZE section to settings panel

**Files:** `MathGaze/MainWindow.xaml`

- After the closing `</StackPanel>` for THEME, add a PROTRACTOR SIZE section with three
  buttons (Small / Medium / Large) using `StepButtonStyle` + `Tag="active"` DataTriggers
  bound to `SettingsVm.IsProtractorSmall`, `IsProtractorMedium`, `IsProtractorLarge`.
- Button height 56px (gaze floor).

### Task 7 — Wrap RightRail main content in ScrollViewer

**Files:** `MathGaze/Views/RightRail.xaml`

- Wrap `<StackPanel Margin="8,14,8,0">` with:
  `<ScrollViewer VerticalScrollBarVisibility="Hidden" HorizontalScrollBarVisibility="Disabled">`
  Grid 3 sends scroll wheel / keyboard events; hidden scrollbar is correct for gaze.

## Commit strategy

Single atomic commit after all 7 tasks.
